# Dify 網路架構關聯說明

本文詳細說明 `dify-ngrok`、`dify-langbot`、`docker-nginx-1` 三個關鍵服務的角色、Port 對應與流量關係。

---

## 三個服務的角色與關係

### 1. docker-nginx-1（反向代理閘道）

Nginx 是 **Dify 平台的統一入口**，負責把外部請求路由到正確的內部服務。

| Host Port | Container Port | 用途 |
|-----------|---------------|------|
| `8080` | `80` | HTTP 主要入口 |
| `443` | `443` | HTTPS 入口 |

**內部路由規則（conf.d/default.conf）：**

| URL 路徑 | 轉發目標 |
|----------|---------|
| `/console/api`, `/api`, `/v1`, `/files`, `/mcp`, `/triggers` | `api:5001` |
| `/`, `/explore` | `web:3000` |
| `/portal`, `/tsc_app/` | `login_app:5050` |
| `/e/` | `plugin_daemon:5002` |

---

### 2. dify-langbot（多平台 AI Bot 平台）

LangBot 是一個整合 Dify 的 **多平台聊天機器人服務**，支援 QQ / Telegram 等平台。

| Host Port | Container Port | 用途 |
|-----------|---------------|------|
| `5300` | `5300` | LangBot Web UI / Dashboard |
| `2280–2290` | `2280–2290` | OneBot v11 協議（QQ/Telegram 等） |

**相依關係：**
- 依賴 `langbot_plugin_runtime` 服務（無對外 Port）
- 可透過 ngrok 暴露到公網

---

### 3. dify-ngrok（公網穿透隧道）

Ngrok 把 **內部服務暴露到公網**，不需要固定 IP 或開防火牆。

| Host Port | Container Port | 用途 |
|-----------|---------------|------|
| `4041` | `4040` | Ngrok 管理 Dashboard |
| — | 公網 HTTPS | 透過 ngrok.io 域名對外暴露 |

**當前 `ngrok.yml` 隧道配置：**
```yaml
version: 2
authtoken: 3CFaIAxP7GCu91270Qdn9BkzqpJ_7s4xWihRTKuPkcfLTX4B7
tunnels:
  langbot:
    proto: http
    addr: dify-langbot:5300   # ← 暴露 LangBot Dashboard 到公網
```

> **注意：** README 原始設計是 ngrok → nginx（暴露整個 Dify），但目前設定暴露的是 LangBot。

## 實際流量路徑

### 1. 本機訪問 Dify 控制台

```
瀏覽器 → localhost:8080 
         → docker-nginx-1:80 
         → web:3000 (前端)
         → api:5001 (API)
```

### 2. 外部用戶透過 ngrok 訪問 LangBot Dashboard

```
外部瀏覽器 → https://xxxx.ngrok.io 
           → dify-ngrok (公網穿透)
           → dify-langbot:5300
```

### 3. 查看 ngrok 管理介面

```
瀏覽器 → localhost:4041 
       → dify-ngrok 內部 Dashboard:4040
       (查看隧道狀態、流量統計等)
```

### 4. QQ / Telegram Bot 連線

```
Bot 平台 → Host:2280-2290 
         → dify-langbot OneBot v11 協議
         → 處理訊息往返
```

## 重點整理

| 服務 | 角色 | 對外入口 | 依賴 |
|------|------|---------|------|
| `docker-nginx-1` | Dify 平台反向代理 | `localhost:8080` | api, web, login_app, plugin_daemon |
| `dify-langbot` | 多平台 Bot 平台 | `localhost:5300` / ngrok 公網 | langbot_plugin_runtime |
| `dify-ngrok` | 公網穿透隧道 | ngrok.io 公網域名 | dify-langbot |

## 配置調整建議

### 若要暴露整個 Dify 平台到公網

修改 `ngrok.yml`：

```yaml
version: 2
authtoken: 3CFaIAxP7GCu91270Qdn9BkzqpJ_7s4xWihRTKuPkcfLTX4B7
tunnels:
  dify:
    proto: http
    addr: nginx:80    # ← 改為 nginx，暴露整個 Dify
```

同時需在 `.env` 中設定 HTTPS Header 支援：

```bash
RESPECT_XFORWARD_HEADERS_ENABLED=true
```

並保持以下欄位為空（讓 Dify 自動判斷域名）：

```bash
CONSOLE_API_URL=
CONSOLE_WEB_URL=
SERVICE_API_URL=
APP_API_URL=
APP_WEB_URL=
FILES_URL=
```

### 常用指令

```bash
# 查看 ngrok 公網 URL 與隧道狀態
curl http://localhost:4041/api/tunnels

# 查看 ngrok 日誌
docker compose logs -f dify-ngrok

# 查看 LangBot 日誌
docker compose logs -f dify-langbot

# 重啟 ngrok 隧道
docker compose restart dify-ngrok
```
