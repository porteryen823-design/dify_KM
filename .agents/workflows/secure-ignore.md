---
description: 確保專案內敏感金鑰與設定檔不被外洩上傳的自動化檢查流程
---
這個工作流將協助您檢查並確保專案中沒有無意間加入 Git 追蹤的敏感資料。每次準備發布或推播到遠端數據庫前，建議執行此指令：

1. **掃描已暫存 (staged) 與未暫存的部分**
搜尋是否包含 `.env`、`.bak`、`.log`、`ngrok.yml`、`keys/` 或其他可能帶有密碼字樣的檔案（如包含 `sk-`, `authtoken`, `secret` 等內容）。

// turbo
2. **自動將敏感檔案加入 .gitignore**
掃描過後，若是發現有未忽略的設定檔，工作流將自動把這些副檔名或指定檔案補寫進專案的 `.gitignore` 之中。

// turbo
3. **退回已加入暫存區的檔案 (Unstage)**
針對不慎執行了 `git add .` 而誤加的敏感檔案，執行 `git restore --staged <file>`。

// turbo 
4. **輸出安全性變更報告**
產生一份詳細報告給使用者確認，指出哪些設定檔剛被移除了追蹤、哪些金鑰字串剛被警告或遮蔽。
