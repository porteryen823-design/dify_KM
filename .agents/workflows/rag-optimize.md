# /rag-optimize 執行步驟

1. **讀取源文件**：利用 `view_file` 讀取使用者指定的原始文檔。
2. **清理與預處理**：
   - 移除文件中不必要的行號（如 `1:`, `2:`）。
   - 統一使用 UTF-8 編碼，將中文標點統整為全形。
3. **孤立標題清除 ⚠️（RAG 品質關鍵步驟）**：
   - 對於 **Q/A 格式文件**，偵測並移除「孤立的 `##` 章節標題行」。
   - **孤立標題定義**：`## 標題` 後方僅跟空白行 + `### Q:` 開頭的內容，將被 Dify 切為無意義描片（如 7 字元的 `繞路與恢復機制`）。
   - **批次處理操作**：執行 `tsc_md_clean.py` （位於 `Doc_tsc/`）進行批次清理。
   - 對於文章格式（無 Q/A 標記），保留標題不作處理。
4. **結構化標註 (Structure)**：
   - 根據文件層級加入 Markdown 標題（`##`, `###`）。
   - 加入 YAML Front-Matter（`title`, `description`, `tags`）。
5. **分段與塊化 (Chunking Optimization)**：
   - 使用 `---` 分隔不相關的語意塊，確保 Dify 切塊時語意完整。
   - 將條列式內容整理為有序或無序列表。
6. **產生目標文件**：
   - 將結果存至使用者指定的目標路徑（如 `xxx.md`）。
7. **回報差異**：描述優化了哪些部分（如格式、標點、結構等）。
