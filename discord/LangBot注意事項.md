# LangBot 使用注意事項：私訊與群組邏輯說明

在使用 LangBot 整合 Discord 或 Telegram 時，機器人的回應行為會根據對話的**場景類型**（私訊或群組）而有所不同。以下整理了核心邏輯與常見問題排除。

## 1. 私訊模式 (Direct Message, DM)
當您直接點擊機器人頭像並發送訊息時，LangBot 會將其視為「私聊模式」。

*   **行為邏輯**：LangBot 預設您發出的每一句話都是要給機器人的。
*   **觸發條件**：**自動忽略**所有「流水線 (Pipeline)」中的觸發規則。
    *   不需要加上 `@提及`。
    *   不需要加上任何 `Prefix`（如 `ai`）。
    *   機器人會對**每一條**訊息進行回應。

## 2. 群組/頻道模式 (Group / Channel)
當機器人被加入到一個 Discord 伺服器頻道或 Telegram 群組時，LangBot 會將其視為「群組模式」。

*   **行為邏輯**：為了避免機器人在多人聊天中不斷插嘴（干擾非 AI 相關的對話），LangBot 會嚴格執行「流水線」中的觸發規則。
*   **觸發條件**：必須符合以下任一條件，流水線才會啟動（根據您的 `AAA` 或 `Telegram Pipeline` 設定）：
    1.  **@提及 (At)**：訊息中必須包含 `@您的機器人名稱`。
    2.  **前綴字 (Prefix)**：訊息開頭必須帶有指定關鍵字（例如 `ai`）。
*   **如果未符合條件**：LangBot 後台雖然會收到 Discord/Telegram 轉發過來的訊息，但會判斷為「非目標對話」而直接**放棄處理 (Drop)**，這就是為什麼您在 Discord 頻道輸入文字卻沒有看到 Dify 流水線被呼叫的原因。

## 3. 為什麼 Telegram 感覺「比較靈敏」？
這通常是因為使用習慣的差異：
*   **Telegram**：用戶通常習慣點擊機器人進行 **1對1 私聊**，因此感覺不用輸入 `ai` 也會回。
*   **Discord**：用戶通常是在 **伺服器頻道 (Channel)** 中測試，即便頻道只有您一個人，Discord API 傳回的仍是 Channel 訊息，被 LangBot 判定為群組模式，所以必須遵守觸發規則。

## 4. 故障排除建議 (Troubleshooting)

如果您發現 Discord 沒反應，請檢查以下兩點：
1.  **改變輸入方式**：嘗試在訊息開頭加上 `ai ` (包含空格) 或 `@機器人`。
2.  **修改後台設定**：
    *   進入 LangBot 後台 -> **流程線** -> 編輯您的 Pipeline (如 `AAA`)。
    *   點選 **觸發條件** -> **Group Respond Rule**。
    *   如果您希望在頻道內不用 `ai` 也能回覆，請嘗試將 `At` 關閉，並移除所有 `Prefix` 內容（請注意這在多人頻道中可能會導致機器人過度回應）。
